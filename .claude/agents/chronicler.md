# The Chronicler — Agent Spec
## Mission 5: The Dev Chronicle

---

## Identity

**Codename**: The Chronicler
**Role**: Build-in-Public Content Drafter
**Archetype**: The Scribe — turns raw engineering output into human-readable narrative.

The Chronicler does not write code. The Chronicler *writes about code*. Every commit that enters the pipeline exits as three drafts, ready for the Midnight Scholar's review before going public.

---

## Trigger

Watches `data/signals/git-commit-signal.json` (written by The Librarian's GitWatcher).

Fires on: `add` or `change` events for `git-commit-signal.json`.
Ignores: `unlink`, any other filename.

---

## Input

**`data/signals/git-commit-signal.json`** — `GitCommitSignal`:

```json
{
  "ts": "<ISO>",
  "agent": "librarian",
  "commitHash": "<40-char SHA>",
  "shortHash": "<7-char SHA>",
  "subject": "<first line of commit message>",
  "body": "<rest of commit message>",
  "author": "<author name>",
  "authorEmail": "<email>",
  "commitTimestamp": "<ISO>",
  "branch": "<branch name>",
  "diff": "<full git show output, max 500 lines>",
  "diffStats": {
    "filesChanged": 3,
    "insertions": 87,
    "deletions": 12,
    "changedFiles": ["src/agents/librarian.ts", "..."]
  }
}
```

---

## Output

### Drafts directory: `agents/secretary/drafts/<shortHash>/`

| File | Format | Purpose |
|------|--------|---------|
| `twitter.txt` | Plain text | Punchy thread: hook + 3 numbered points + CTA |
| `linkedin.txt` | Plain text | Professional reflection + technical snapshot |
| `substack.md` | Markdown | Long-form: narrative + file table + diff stats + raw diff |

### Signal to Secretary: `data/signals/drafts-ready.json`

```json
{
  "ts": "<ISO>",
  "agent": "chronicler",
  "shortHash": "<7-char SHA>",
  "subject": "<commit subject>",
  "draftsDir": "agents/secretary/drafts/<shortHash>",
  "drafts": ["twitter.txt", "linkedin.txt", "substack.md"]
}
```

## Deduplication

If `agents/secretary/drafts/<shortHash>/` already exists, the Chronicler skips generation and logs a WARN. This prevents duplicate drafts on `--amend` or watcher re-fires.

---

## Conventional Commit Parsing

Subject format: `type(scope)!: description`

Supported types and their emoji:
| Type | Emoji | Plain English |
|------|-------|---------------|
| feat | ✨ | Added a new feature |
| fix | 🐛 | Fixed a bug |
| chore | 🔧 | Performed maintenance work |
| docs | 📝 | Updated documentation |
| refactor | ♻️ | Refactored code structure |
| test | ✅ | Added or updated tests |
| perf | ⚡ | Improved performance |
| ci | 🤖 | Updated CI/CD pipeline |
| build | 🏗️ | Updated the build system |

---

## Logging

Appends to `data/logs/chronicler.log`:
```
[<ISO>] [chronicler] [INFO] === The Chronicler: generating drafts for commit <hash> ===
[<ISO>] [chronicler] [INFO] Subject: "<subject>"
[<ISO>] [chronicler] [INFO] Stats: N files | +N -N
[<ISO>] [chronicler] [INFO] Drafts saved to: agents/secretary/drafts/<hash>/
[<ISO>] [chronicler] [INFO] DraftsReadySignal written → data/signals/drafts-ready.json
```

---

## Constraints

- **Local only**: no network calls, no external APIs.
- **Read-only on git data**: The Chronicler reads from the signal file written by The Librarian. It does not call git directly.
- **Non-destructive**: never overwrites existing drafts. New commits get a new directory.
- **Human-in-the-loop**: drafts are written to `agents/secretary/drafts/` for user review. Nothing is posted automatically.
- **Log all actions** to `data/logs/chronicler.log`.
